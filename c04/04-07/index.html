    redis cluster-场景测试

    测试准备

    硬件详情

    * 服务器详情：双核，CentOS 6.5
* redis: 三对主从redis[(7000,7003),(7001,7004),(7002,7005)]

    压测工具

    * 压测命令：redis-benchmark

    几种常见场景

    * 使用redis-benchmark进行压测，查看压测结果
* 压测中，主从切换，是否影响业务
* 压测中，重新分片，是否影响业务

    用到的命令

    • 重新分片
    redis-trib.rb reshard <host>:<port>
redis-trib.rb check 127.0.0.1:7000
redis-trib.rb fix <host>:<port>
    • 压测 1000并发，总数为100000
    redis-benchmark -h 10.1.249.172 -p 7000 -d 100 -c 1000 -n 100000 -q -t get,set &
    • 客户端连接
    redis-cli -h 10.1.249.172 -p 7000 CLUSTER GETKEYSINSLOT 1179 999
redis-cli -h 10.1.249.172 -p 7000 CLUSTER KEYSLOT myzset1
redis-cli -h 10.1.249.172 -p 7004 CLUSTER setslot  1179 stable
redis-cli -h 127.0.0.1 -p 7003 -c
redis-cli -h 127.0.0.1 -p 7003 cluster nodes

    测试结果

    redis-benchmark 压测结果

    压测实例数 读（GET） 写（SET）
    1个实例(1000并发） 38729 37169
    2个实例(500并发） 65653 49382
    4个实例(250并发） 85890 82042

    压测中，主从切换

    步骤：
1. redis-benchmark 压测中
2. redis主从关系：[(7000,7003),(7001,7004),(7002,7005)]
3. 控制页面上， 停止7000，7003变成master，然后让7000重新运行。主从关系由“7000,7003” 变成“7003,7000” 
4. 使用api调用redis，读和写key正常

结果：
压测中，主从切换，对业务无影响

    压测中，重新分片

    步骤：
1. redis-benchmark 压测中
2. 使用api往一个key中，批量写入海量数据
3. 使用 redis-trib.rb reshard 重新分片，主要移动该key所在的slot
4. 使用api调用redis，读和写key正常

结果：
压测中，重新分片，对业务无影响

注意点：
1. 重新分片，如果单个slot中的key体积太大（占用内存2GB左右），很容易超时失败。可以用redis-trib.rb fix <host>:<port>修复。
2. 重新分片，超时失败的话，对redis集群的读取key没有影响。
