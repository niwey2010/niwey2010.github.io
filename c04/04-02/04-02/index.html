    Nexus 3.0（仓库管理 ）

    由于网络和nexus本地缓存的原因，建议使用nexus源。

    1. docker使用nexus私有源

    1.1 nexus私有源

    拉取docker服务器："192.168.0.1:8082" 
提交docker服务器："192.168.0.1:8083"

    1.2 因为使用的https证书是自签名，需要手动添加忽略https安全验证列表。

    • **当系统是windows 10，"insecure-registries" 节点下需要添加过滤
    {
  "insecure-registries": [
    "192.168.0.1:8082",
    "192.168.0.1:8083"
  ]
}
    • 当系统是其他系统时。 官方文档docker insecure
    1. 当系统是linux时，修改 /etc/docker/daemon.json (文件若不存在，可新建)
{
  "insecure-registries": [
    "192.168.0.1:8082",
    "192.168.0.1:8083"
  ],
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com/"
  ]
}

2. 重启服务
systemctl restart docker && systemctl status docker -l

    1.3 拉取镜像（服务器:192.168.0.1:8082）

    # 登录192.168.0.1:8082（只需登录成功一次即可）
docker login 192.168.0.1:8082 

# 搜索镜像
docker search 192.168.0.1:8082/hello-world

# 拉取镜像
docker pull 192.168.0.1:8082/hello-world

    1.4 上传自定义镜像（服务器:192.168.0.1:8083）

    # 拉取镜像
docker pull 192.168.0.1:8082/hello-world

# 运行
docker run -t -i 192.168.0.1:8082/centos

# 查看任务运行，在docker commit时需要其中的<CONTAINER ID>
docker ps -a 

# 提交
docker commit -m "test" <CONTAINER ID> 192.168.0.1:8083/nexus-test

# 登录192.168.0.1:8083（只需登录成功一次即可）
docker login 192.168.0.1:8083

# 推送本地自定义的镜像
docker push 192.168.0.1:8083/nexus-test

# 尝试拉取刚上传过的镜像，应该可以成功拉取
docker pull 192.168.0.1:8082/nexus-test

    2. maven使用nexus源

    2.1 项目使用本地仓库

    • 修改build.gradle，改为本地仓库组:
        repositories {
        maven {
            url "http://192.168.0.1:8081/repository/maven-public/"
        }
    }

    2.2 上传本地项目jar到私有库releases和snapshots

    • 修改build.gradle, 然后在项目根目录下执行"gradle upload"，即可上传成功。
    apply plugin: 'maven'
uploadArchives {
    repositories {
        mavenDeployer {
            repository(url: "http://192.168.0.1:8081/repository/maven-releases")
                    {
                        authentication(userName: 'tester', password: 'tester')
                    }
            snapshotRepository(url: "http://192.168.0.1:8081/repository/maven-snapshots")
                    {
                authentication(userName: 'tester', password: 'tester')
                    }
            pom.version = "1.0.1"
            pom.artifactId = "example"
            pom.groupId = "com.xxx"
        }
    }
}
    • 备注：版本号设置。若版本号以"-SNAPSHOT"结尾,会发布到release仓库，其他情况发布到releases分支
    pom.version = "1.0.1-SNAPSHOT"

    3.0 官方文档

    传送门

    3.1 项目使用本地仓库

    • 操作系统是windows，新增配置文件(C:\Users\<你的帐户名>\pip\pip.ini)。
    [global]
index = http://192.168.0.1:8081/repository/pypi-public/pypi
index-url = http://192.168.0.1:8081/repository/pypi-public/simple

[install]
trusted-host=192.168.0.1
    • 其他操作系统

    TBC

    3.2 上传本地项目到私有库releases

    • 操作系统是windows，新建文件（C:\Users\<你的帐户名>.pypirc）
    [distutils]
index-servers = nexus

[nexus]
repository = http://192.168.0.1:8081/repository/pypi-releases/
username = tester
password = tester

    6. nexus 自签名证书

    官方帮助链接

    keytool -genkeypair -keystore keystore.jks -storepass nexus3 -keypass nexus3 -alias jetty -keyalg RSA -keysize 2048 -validity 5000 -dname "CN=*.nexus, OU=Example, O=Sonatype, L=Unspecified, ST=Unspecified, C=US" -ext "SAN=DNS:nexus,IP:192.168.0.1" -ext "BC=ca:true"

keytool -delete -keystore keystore.jks -alias jetty
